## Prerequisites

- **Node.js >= 22.12.0** â€” the `engines` field in `package.json` enforces this version.
- A running Academia Gema REST API backend with its base URL available.
- `git` installed on your machine.

## Local setup

<Steps>
  <Step title="Clone the repository">
    Clone the project and navigate into the frontend directory.

    ```bash
    git clone <your-repo-url>
    cd <repo-root>/Front
    ```
  </Step>

  <Step title="Install dependencies">
    Install all npm dependencies declared in `package.json`.

    <CodeGroup>
      ```bash npm
      npm install
      ```
      ```bash yarn
      yarn install
      ```
    </CodeGroup>

    Key runtime dependencies installed:

    | Package | Version | Purpose |
    |---|---|---|
    | `react` | ^19.2.0 | UI framework |
    | `react-router-dom` | ^7.12.0 | Client-side routing |
    | `framer-motion` | ^12.34.0 | Animations |
    | `recharts` | ^3.8.0 | Dashboard charts |
    | `js-cookie` | ^3.0.5 | JWT cookie access |
    | `xlsx` / `xlsx-js-style` | ^0.18.5 / ^1.2.0 | Excel exports |
  </Step>

  <Step title="Configure the environment variable">
    Create a `.env` file in the `Front/` directory and set the backend API URL:

    ```bash .env
    VITE_API_URL=https://your-api-hostname.com
    ```

    <Warning>
      Do **not** include a trailing slash. The interceptor appends endpoint paths directly, e.g. `${VITE_API_URL}/auth/login`.
    </Warning>

    The variable is read at build time by Vite and accessed in the code via `import.meta.env.VITE_API_URL`. Only variables prefixed with `VITE_` are exposed to the browser bundle.
  </Step>

  <Step title="TailwindCSS (already configured)">
    TailwindCSS 3 is fully configured and ready to use. No additional setup is needed â€” the configuration is already in place:

    ```js tailwind.config.js
    /** @type {import('tailwindcss').Config} */
    export default {
      content: [
        "./index.html",
        "./src/**/*.{js,ts,jsx,tsx}",
      ],
      theme: {
        extend: {},
      },
      plugins: [],
    }
    ```

    PostCSS and Autoprefixer are also pre-configured via `postcss.config.js`.
  </Step>

  <Step title="Start the development server">
    Launch Vite's dev server with hot-module replacement:

    <CodeGroup>
      ```bash npm
      npm run dev
      ```
      ```bash yarn
      yarn dev
      ```
    </CodeGroup>

    Vite will start on `http://localhost:5173` by default and print the local URL to the terminal.

    <Tip>
      The Vite config sets `server.allowedHosts: true`, so you can expose the dev server to other devices on your network without additional configuration.
    </Tip>
  </Step>

  <Step title="Log in to the application">
    Open your browser to `http://localhost:5173/login`. Enter credentials for any of the three roles:

    - **Administrador** â€” redirected to `/dashboard/admin`
    - **Coordinador** â€” redirected to `/dashboard/teacher`
    - **Alumno** â€” redirected to `/dashboard/student`

    The auth flow stores the JWT in an HTTP-only cookie managed server-side. `js-cookie` reads the `user_role`, `user_name`, and `user_id` cookies that the API sets alongside the token so the frontend can route and render role-specific UI without decoding the JWT.
  </Step>
</Steps>

## Available npm scripts

```json package.json (scripts)
"scripts": {
  "dev":     "vite",
  "build":   "vite build",
  "lint":    "eslint .",
  "preview": "vite preview --host 0.0.0.0 --port ${PORT:-4173}",
  "start":   "npm run preview"
}
```

| Script | Purpose |
|---|---|
| `dev` | Local development server with HMR |
| `build` | Outputs production bundle to `dist/` |
| `preview` / `start` | Serves the `dist/` bundle locally, used by Railway |
| `lint` | Runs ESLint with React Hooks and React Refresh plugins |

## Deployment

<CardGroup cols={2}>
  <Card title="Netlify" icon="cloud">
    A `netlify.toml` is included at the project root. Netlify runs `npm run build`, publishes the `dist/` directory, and handles SPA redirects automatically.

    ```toml netlify.toml
    [build]
      base    = ""
      publish = "dist"
      command = "npm run build"
    ```
  </Card>
  <Card title="Railway" icon="train">
    A `railway.json` is included. Railway builds the project and starts it with `npm run start` (which runs `vite preview`). Set the `PORT` environment variable in your Railway service â€” the preview script reads it as `${PORT:-4173}`.
  </Card>
</CardGroup>

<Note>
  Set `VITE_API_URL` as an environment variable in your deployment platform's settings â€” do not commit `.env` to version control.
</Note>
