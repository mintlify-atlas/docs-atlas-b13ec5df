title: "Database schema"
description: "Complete reference for all tables in the Academia Gema PostgreSQL database, grouped by functional domain."
---

The Academia Gema database is a **PostgreSQL** schema defined in `BD/clubGema.sql`. Tables are organised into five functional groups, each building on the previous one.

<Note>
  All timestamp columns that track the last update time are automatically maintained by the `trg_actualizar_*` triggers. You do not need to set `actualizado_en` manually on `UPDATE` statements.
</Note>

## Group 1 — Roles & Users

<AccordionGroup>
  <Accordion title="roles — Role catalog">
    Stores the allowed roles for the platform. Every user is assigned exactly one role.

    | Column | Type | Constraints |
    |---|---|---|
    | `id` | `SERIAL` | Primary key |
    | `nombre` | `VARCHAR(50)` | `UNIQUE NOT NULL` |

    ```sql
    CREATE TABLE roles (
        id SERIAL PRIMARY KEY,
        nombre VARCHAR(50) UNIQUE NOT NULL -- e.g. 'ADMIN', 'PROFESOR', 'ALUMNO'
    );
    ```
  </Accordion>

  <Accordion title="usuarios — Core user accounts">
    Central user table. All role-specific profiles extend this table via foreign key on `id`.

    | Column | Type | Constraints | Notes |
    |---|---|---|---|
    | `id` | `SERIAL` | Primary key | |
    | `rol_id` | `INT` | `NOT NULL`, FK → `roles(id)` | Determines the role |
    | `nombres` | `VARCHAR(100)` | `NOT NULL` | First name(s) |
    | `apellidos` | `VARCHAR(100)` | `NOT NULL` | Last name(s) |
    | `email` | `VARCHAR(150)` | `UNIQUE NOT NULL` | Login identifier |
    | `telefono` | `VARCHAR(20)` | | Personal phone |
    | `activo` | `BOOLEAN` | `DEFAULT TRUE` | Soft delete flag |
    | `creado_en` | `TIMESTAMP` | `DEFAULT CURRENT_TIMESTAMP` | |
    | `actualizado_en` | `TIMESTAMP` | `DEFAULT CURRENT_TIMESTAMP` | Auto-updated by trigger |

    ```sql
    CREATE TABLE usuarios (
        id SERIAL PRIMARY KEY,
        rol_id INT NOT NULL REFERENCES roles(id),
        nombres VARCHAR(100) NOT NULL,
        apellidos VARCHAR(100) NOT NULL,
        email VARCHAR(150) UNIQUE NOT NULL,
        telefono VARCHAR(20),
        activo BOOLEAN DEFAULT TRUE,
        creado_en TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
        actualizado_en TIMESTAMP DEFAULT CURRENT_TIMESTAMP
    );
    ```

    **Indexes:** `idx_usuarios_email` on `(email)`, `idx_usuarios_apellidos` on `(apellidos)`.
  </Accordion>

  <Accordion title="credenciales_usuario — Credentials">
    Separated from `usuarios` for security. Stores hashed passwords, login tracking, and brute-force protection.

    | Column | Type | Constraints | Notes |
    |---|---|---|---|
    | `usuario_id` | `INT` | Primary key, FK → `usuarios(id)` ON DELETE CASCADE | |
    | `hash_contrasena` | `VARCHAR(255)` | `NOT NULL` | bcrypt hash |
    | `ultimo_login` | `TIMESTAMP` | | Last successful login |
    | `intentos_fallidos` | `INT` | `DEFAULT 0` | Increments on wrong password |
    | `bloqueado` | `BOOLEAN` | `DEFAULT FALSE` | Account lock flag |
    | `contrasena_actualizada_en` | `TIMESTAMP` | `DEFAULT CURRENT_TIMESTAMP` | |

    ```sql
    CREATE TABLE credenciales_usuario (
        usuario_id INT PRIMARY KEY REFERENCES usuarios(id) ON DELETE CASCADE,
        hash_contrasena VARCHAR(255) NOT NULL,
        ultimo_login TIMESTAMP,
        intentos_fallidos INT DEFAULT 0,
        bloqueado BOOLEAN DEFAULT FALSE,
        contrasena_actualizada_en TIMESTAMP DEFAULT CURRENT_TIMESTAMP
    );
    ```
  </Accordion>
</AccordionGroup>

## Group 2 — Profiles

<AccordionGroup>
  <Accordion title="alumnos — Student profiles">
    Extends `usuarios` with student-specific fields. One-to-one relationship with `usuarios`.

    | Column | Type | Constraints | Notes |
    |---|---|---|---|
    | `usuario_id` | `INT` | Primary key, FK → `usuarios(id)` | |
    | `contacto_emergencia_nombre` | `VARCHAR(100)` | | Emergency contact name |
    | `contacto_emergencia_telefono` | `VARCHAR(20)` | | Emergency contact phone |
    | `condiciones_medicas` | `TEXT` | | Free-text medical notes |
    | `fecha_nacimiento` | `DATE` | | Used for age-range analytics |

    ```sql
    CREATE TABLE alumnos (
        usuario_id INT PRIMARY KEY REFERENCES usuarios(id),
        contacto_emergencia_nombre VARCHAR(100),
        contacto_emergencia_telefono VARCHAR(20),
        condiciones_medicas TEXT,
        fecha_nacimiento DATE
    );
    ```
  </Accordion>

  <Accordion title="profesores — Coordinator profiles">
    Extends `usuarios` for coordinators. The `tarifa_hora` field is used for payroll calculations.

    | Column | Type | Constraints | Notes |
    |---|---|---|---|
    | `usuario_id` | `INT` | Primary key, FK → `usuarios(id)` | |
    | `especializacion` | `VARCHAR(100)` | | Volleyball specialisation (e.g. blocking, setting) |
    | `tarifa_hora` | `DECIMAL(10,2)` | | Hourly rate in PEN |

    ```sql
    CREATE TABLE profesores (
        usuario_id INT PRIMARY KEY REFERENCES usuarios(id),
        especializacion VARCHAR(100),
        tarifa_hora DECIMAL(10, 2)
    );
    ```
  </Accordion>
</AccordionGroup>

## Group 3 — Infrastructure & Academic

<AccordionGroup>
  <Accordion title="direcciones — Normalised addresses">
    Stores physical addresses in a normalised form. Referenced by `sedes`.

    | Column | Type | Notes |
    |---|---|---|
    | `id` | `SERIAL PK` | |
    | `calle_principal` | `VARCHAR(255) NOT NULL` | Main street |
    | `calle_secundaria` | `VARCHAR(255)` | Intersection or reference |
    | `ciudad` | `VARCHAR(100) NOT NULL` | City |
    | `estado_provincia` | `VARCHAR(100)` | Department / Province |
    | `codigo_postal` | `VARCHAR(20)` | |
    | `pais` | `VARCHAR(50)` | `DEFAULT 'Peru'` |
    | `creado_en` | `TIMESTAMP` | `DEFAULT CURRENT_TIMESTAMP` |

    ```sql
    CREATE TABLE direcciones (
        id SERIAL PRIMARY KEY,
        calle_principal VARCHAR(255) NOT NULL,
        calle_secundaria VARCHAR(255),
        ciudad VARCHAR(100) NOT NULL,
        estado_provincia VARCHAR(100),
        codigo_postal VARCHAR(20),
        pais VARCHAR(50) DEFAULT 'Peru',
        creado_en TIMESTAMP DEFAULT CURRENT_TIMESTAMP
    );
    ```

    **Index:** `idx_direcciones_ciudad` on `(ciudad)`.
  </Accordion>

  <Accordion title="sedes — Club locations">
    Physical club locations (branches). Each sede references one `direcciones` record.

    | Column | Type | Notes |
    |---|---|---|
    | `id` | `SERIAL PK` | |
    | `direccion_id` | `INT NOT NULL` | FK → `direcciones(id)` |
    | `nombre` | `VARCHAR(100) NOT NULL` | e.g. `'Sede Norte'` |
    | `activo` | `BOOLEAN` | `DEFAULT TRUE` |

    ```sql
    CREATE TABLE sedes (
        id SERIAL PRIMARY KEY,
        direccion_id INT NOT NULL REFERENCES direcciones(id),
        nombre VARCHAR(100) NOT NULL,
        activo BOOLEAN DEFAULT TRUE
    );
    ```
  </Accordion>

  <Accordion title="canchas — Courts">
    Volleyball courts within a sede. Used to enforce no double-booking via the unique constraint on `horarios_clases`.

    | Column | Type | Notes |
    |---|---|---|
    | `id` | `SERIAL PK` | |
    | `sede_id` | `INT NOT NULL` | FK → `sedes(id)` |
    | `nombre` | `VARCHAR(50) NOT NULL` | e.g. `'Cancha 1 - Techada'` |
    | `descripcion` | `VARCHAR(200)` | |

    ```sql
    CREATE TABLE canchas (
        id SERIAL PRIMARY KEY,
        sede_id INT NOT NULL REFERENCES sedes(id),
        nombre VARCHAR(50) NOT NULL,
        descripcion VARCHAR(200)
    );
    ```
  </Accordion>

  <Accordion title="niveles_entrenamiento — Training levels">
    Defines the training levels offered (e.g. Principiante, Intermedio, Avanzado). Each level has a fixed monthly price that feeds the subscription catalog.

    | Column | Type | Notes |
    |---|---|---|
    | `id` | `SERIAL PK` | |
    | `nombre` | `VARCHAR(50) NOT NULL` | Level name |
    | `precio_mensual` | `DECIMAL(10,2) NOT NULL` | Monthly fee in PEN |

    ```sql
    CREATE TABLE niveles_entrenamiento (
        id SERIAL PRIMARY KEY,
        nombre VARCHAR(50) NOT NULL,
        precio_mensual DECIMAL(10, 2) NOT NULL
    );
    ```
  </Accordion>

  <Accordion title="horarios_clases — Class schedules">
    The central scheduling table. Each row represents a recurring weekly class slot. The unique constraint prevents double-booking a court at the same day and time.

    | Column | Type | Notes |
    |---|---|---|
    | `id` | `SERIAL PK` | |
    | `sede_id` | `INT NOT NULL` | FK → `sedes(id)` |
    | `cancha_id` | `INT` | FK → `canchas(id)` |
    | `profesor_id` | `INT NOT NULL` | FK → `profesores(usuario_id)` |
    | `nivel_id` | `INT NOT NULL` | FK → `niveles_entrenamiento(id)` |
    | `dia_semana` | `INT NOT NULL` | `CHECK (1..7)`, 1 = Monday |
    | `hora_inicio` | `TIME NOT NULL` | |
    | `hora_fin` | `TIME NOT NULL` | |
    | `capacidad_max` | `INT` | `DEFAULT 20` |
    | `activo` | `BOOLEAN` | `DEFAULT TRUE` |

    **Constraint:** `UNIQUE (cancha_id, dia_semana, hora_inicio)` — prevents two classes from sharing the same court at the same time.

    ```sql
    CREATE TABLE horarios_clases (
        id SERIAL PRIMARY KEY,
        sede_id INT NOT NULL REFERENCES sedes(id),
        cancha_id INT REFERENCES canchas(id),
        profesor_id INT NOT NULL REFERENCES profesores(usuario_id),
        nivel_id INT NOT NULL REFERENCES niveles_entrenamiento(id),
        dia_semana INT NOT NULL CHECK (dia_semana BETWEEN 1 AND 7),
        hora_inicio TIME NOT NULL,
        hora_fin TIME NOT NULL,
        capacidad_max INT DEFAULT 20,
        activo BOOLEAN DEFAULT TRUE,
        CONSTRAINT unica_cancha_hora UNIQUE (cancha_id, dia_semana, hora_inicio)
    );
    ```
  </Accordion>
</AccordionGroup>

## Group 4 — Operations

<AccordionGroup>
  <Accordion title="inscripciones — Enrollments">
    Records which student is enrolled in which class schedule. The unique constraint prevents a student from enrolling in the same class twice.

    | Column | Type | Notes |
    |---|---|---|
    | `id` | `SERIAL PK` | |
    | `alumno_id` | `INT NOT NULL` | FK → `alumnos(usuario_id)` |
    | `horario_id` | `INT NOT NULL` | FK → `horarios_clases(id)` |
    | `fecha_inscripcion` | `DATE` | `DEFAULT CURRENT_DATE` |
    | `estado` | `VARCHAR(20)` | `'ACTIVO'` \| `'SUSPENDIDO'` \| `'RETIRADO'` |
    | `actualizado_en` | `TIMESTAMP` | Auto-updated by trigger |

    **Constraint:** `UNIQUE (alumno_id, horario_id)` — one enrollment per student per schedule.

    **Indexes:** `idx_inscripciones_alumno` on `(alumno_id)`, `idx_inscripciones_horario` on `(horario_id)`.

    ```sql
    CREATE TABLE inscripciones (
        id SERIAL PRIMARY KEY,
        alumno_id INT NOT NULL REFERENCES alumnos(usuario_id),
        horario_id INT NOT NULL REFERENCES horarios_clases(id),
        fecha_inscripcion DATE DEFAULT CURRENT_DATE,
        estado VARCHAR(20) DEFAULT 'ACTIVO' CHECK (estado IN ('ACTIVO', 'SUSPENDIDO', 'RETIRADO')),
        actualizado_en TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
        CONSTRAINT unica_inscripcion_alumno UNIQUE (alumno_id, horario_id)
    );
    ```
  </Accordion>

  <Accordion title="registros_asistencia — Attendance records">
    One row per student per session date. The unique constraint ensures a student can only have one attendance record per enrollment per day.

    | Column | Type | Notes |
    |---|---|---|
    | `id` | `SERIAL PK` | |
    | `inscripcion_id` | `INT NOT NULL` | FK → `inscripciones(id)` |
    | `fecha` | `DATE NOT NULL` | Session date |
    | `estado` | `VARCHAR(20)` | `'PRESENTE'` \| `'AUSENTE'` \| `'TARDE'` \| `'JUSTIFICADO'` |
    | `registrado_en` | `TIMESTAMP` | `DEFAULT CURRENT_TIMESTAMP` |

    **Constraint:** `UNIQUE (inscripcion_id, fecha)` — one record per enrollment per day.

    **Index:** `idx_asistencia_inscripcion` on `(inscripcion_id)`.

    ```sql
    CREATE TABLE registros_asistencia (
        id SERIAL PRIMARY KEY,
        inscripcion_id INT NOT NULL REFERENCES inscripciones(id),
        fecha DATE NOT NULL,
        estado VARCHAR(20) DEFAULT 'PRESENTE' CHECK (estado IN ('PRESENTE', 'AUSENTE', 'TARDE', 'JUSTIFICADO')),
        registrado_en TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
        CONSTRAINT asistencia_unica_diaria UNIQUE (inscripcion_id, fecha)
    );
    ```
  </Accordion>
</AccordionGroup>

## Group 5 — Finance

<AccordionGroup>
  <Accordion title="cuentas_por_cobrar — Accounts receivable">
    Represents a charge issued to a student — monthly fees, penalties, or other concepts. Each payment is linked to one of these records.

    | Column | Type | Notes |
    |---|---|---|
    | `id` | `SERIAL PK` | |
    | `alumno_id` | `INT NOT NULL` | FK → `alumnos(usuario_id)` |
    | `concepto` | `VARCHAR(150) NOT NULL` | Billing concept description |
    | `monto` | `DECIMAL(10,2) NOT NULL` | Amount charged in PEN |
    | `fecha_vencimiento` | `DATE NOT NULL` | Payment due date |
    | `estado` | `VARCHAR(20)` | `'PENDIENTE'` \| `'PARCIAL'` \| `'PAGADO'` \| `'VENCIDO'` \| `'CANCELADO'` |
    | `creado_en` | `TIMESTAMP` | `DEFAULT CURRENT_TIMESTAMP` |
    | `actualizado_en` | `TIMESTAMP` | Auto-updated by trigger |

    **Indexes:** `idx_cuentas_alumno` on `(alumno_id)`, `idx_cuentas_estado_fecha` on `(estado, fecha_vencimiento)`.

    ```sql
    CREATE TABLE cuentas_por_cobrar (
        id SERIAL PRIMARY KEY,
        alumno_id INT NOT NULL REFERENCES alumnos(usuario_id),
        concepto VARCHAR(150) NOT NULL,
        monto DECIMAL(10, 2) NOT NULL,
        fecha_vencimiento DATE NOT NULL,
        estado VARCHAR(20) DEFAULT 'PENDIENTE' CHECK (estado IN ('PENDIENTE', 'PARCIAL', 'PAGADO', 'VENCIDO', 'CANCELADO')),
        creado_en TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
        actualizado_en TIMESTAMP DEFAULT CURRENT_TIMESTAMP
    );
    ```
  </Accordion>

  <Accordion title="pagos — Payments">
    Records each individual payment transaction against an account receivable. A single `cuentas_por_cobrar` record can have multiple `pagos` rows (partial payments).

    | Column | Type | Notes |
    |---|---|---|
    | `id` | `SERIAL PK` | |
    | `cuenta_id` | `INT NOT NULL` | FK → `cuentas_por_cobrar(id)` |
    | `monto_pagado` | `DECIMAL(10,2) NOT NULL` | Actual amount paid |
    | `fecha_pago` | `TIMESTAMP` | `DEFAULT CURRENT_TIMESTAMP` |
    | `metodo_pago` | `VARCHAR(50) NOT NULL` | `YAPE`, `PLIN`, `TRANSFERENCIA`, `EFECTIVO` |
    | `referencia_transaccion` | `VARCHAR(100)` | Yape/Plin code or bank reference |

    ```sql
    CREATE TABLE pagos (
        id SERIAL PRIMARY KEY,
        cuenta_id INT NOT NULL REFERENCES cuentas_por_cobrar(id),
        monto_pagado DECIMAL(10, 2) NOT NULL,
        fecha_pago TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
        metodo_pago VARCHAR(50) NOT NULL,
        referencia_transaccion VARCHAR(100)
    );
    ```
  </Accordion>
</AccordionGroup>

## Trigger — auto-updating timestamps

A shared PL/pgSQL function and three triggers keep `actualizado_en` columns current automatically:

```sql
CREATE OR REPLACE FUNCTION actualizar_columna_timestamp()
RETURNS TRIGGER AS $$
BEGIN
    NEW.actualizado_en = NOW();
    RETURN NEW;
END;
$$ language 'plpgsql';

CREATE TRIGGER trg_actualizar_usuarios
    BEFORE UPDATE ON usuarios
    FOR EACH ROW EXECUTE PROCEDURE actualizar_columna_timestamp();

CREATE TRIGGER trg_actualizar_cuentas
    BEFORE UPDATE ON cuentas_por_cobrar
    FOR EACH ROW EXECUTE PROCEDURE actualizar_columna_timestamp();

CREATE TRIGGER trg_actualizar_inscripciones
    BEFORE UPDATE ON inscripciones
    FOR EACH ROW EXECUTE PROCEDURE actualizar_columna_timestamp();
```

<Tip>
  The same function is reused by all three triggers — if you add a new table with an `actualizado_en` column, simply create a new `BEFORE UPDATE` trigger pointing to `actualizar_columna_timestamp()`.
</Tip>
