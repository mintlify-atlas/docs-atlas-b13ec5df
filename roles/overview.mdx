Academia Gema uses three distinct roles to separate concerns across the platform. Each role maps to a dedicated dashboard layout, a set of protected routes, and a specific subset of API endpoints.

## Role definitions

Roles are stored in the `roles` database table with a simple structure:

```sql
CREATE TABLE roles (
    id SERIAL PRIMARY KEY,
    nombre VARCHAR(50) UNIQUE NOT NULL -- e.g. 'ADMIN', 'PROFESOR', 'ALUMNO'
);
```

Every user in `usuarios` holds a foreign key to this table via `rol_id`.

## Allowed role strings per group

The front-end router accepts multiple string variants for each role to maintain backwards compatibility across API versions:

| Group | Accepted role strings | Dashboard path |
|---|---|---|
| Student | `Alumno`, `student`, `alumno` | `/dashboard/student` |
| Coordinator | `Coordinador`, `teacher`, `profesor` | `/dashboard/teacher` |
| Administrator | `Administrador`, `admin`, `administrador` | `/dashboard/admin` |

## How ProtectedRoute works

Every authenticated section of the app is wrapped by the `ProtectedRoute` component (`src/components/ProtectedRoute.jsx`). It performs two checks on every render:

<Steps>
  <Step title="Authentication check">
    Reads the current user from `AuthContext`. If no user is found it redirects immediately to `/login`.
  </Step>
  <Step title="Role authorisation check">
    Reads the role from `user.rol`, `user.user.rol`, or the `user_role` cookie as a fallback. If the role is not in the `allowedRoles` array, the user is redirected to `/forbidden`.
  </Step>
  <Step title="Render outlet">
    If both checks pass, React Router's `<Outlet />` is rendered, mounting the appropriate layout and child pages.
  </Step>
</Steps>

```jsx
// src/components/ProtectedRoute.jsx
const ProtectedRoute = ({ allowedRoles }) => {
    const { user, loading } = useAuth();

    if (loading) return <LoadingSpinner />;

    const isAuthenticated = !!user;
    if (!isAuthenticated) return <Navigate to="/login" replace />;

    const currentRole = user?.rol || user?.user?.rol || Cookies.get('user_role');

    if (allowedRoles && (!currentRole || !allowedRoles.includes(currentRole))) {
        return <Navigate to="/forbidden" replace />;
    }

    return <Outlet />;
};
```

In `App.jsx` each group is declared as:

```jsx
// Student group
<Route element={<ProtectedRoute allowedRoles={['Alumno', 'student', 'alumno']} />}>
  <Route element={<StudentLayout />}>
    <Route path="student" element={<DashboardEstudiante />} />
    {/* … */}
  </Route>
</Route>

// Coordinator group
<Route element={<ProtectedRoute allowedRoles={['Coordinador', 'teacher', 'profesor']} />}>
  <Route element={<TeacherLayout />}>
    <Route path="teacher" element={<DashboardTeacher />} />
    {/* … */}
  </Route>
</Route>

// Administrator group
<Route element={<ProtectedRoute allowedRoles={['Administrador', 'admin', 'administrador']} />}>
  <Route element={<DashboardLayout />}>
    <Route path="admin" element={<Dashboard role="admin" />} />
    {/* … */}
  </Route>
</Route>
```
